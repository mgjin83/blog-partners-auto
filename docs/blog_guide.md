# 블로그팀 가이드 (Blog Guide)

## 읽어야 할 파일
- CLAUDE.md
- 이 파일 (blog_guide.md)
- bible/blog_voice.md
- learnings/ (블로그 관련 항목)

## 리서처 (Blog Researcher)

### 역할
레퍼런스 블로거를 분석하여 도플갱어 톤 가이드를 만든다.

### 작업 흐름
1. 레퍼런스 블로거 URL을 받는다
2. 글 구조, 말투, 제목 패턴, 이미지 배치, SEO 전략을 분석한다
3. 산출물: bible/blog_voice.md 초안 (인간 승인 후 bible에 등록)

### 성공 기준
- 분석 결과로 라이터가 바로 글을 쓸 수 있다
- 톤/구조/패턴이 구체적 예시와 함께 설명되어 있다

## 라이터 (Blog Writer)

### 역할
도플갱어 톤으로 파트너스 글을 작성한다.

### 작업 흐름
1. blog_voice.md를 읽는다
2. 키워드를 받는다
3. 도플갱어 톤으로 글을 작성한다
4. 산출물: blog/naver/ 또는 blog/wordpress/에 글 파일

### 성공 기준
- blog_voice.md의 톤/구조와 일치한다
- 레퍼런스 블로거의 글과 나란히 놓았을 때 자연스럽다
- 파트너스 링크가 삽입될 위치가 명확하다
- 검색 상위 노출에 유리한 키워드 배치가 되어 있다

## 블로그 운영 전략
- 네이버 블로그: 한 계정에 여러 니치를 섞어서 매일 글 생산. 블로그 지수를 올린다.
- 워드프레스: "best ○○ 2026" 류의 비교/추천 글 중심.
- 어떤 니치가 상위 노출 + 전환이 잘 되는지 데이터를 수집한다.
- 이미지 생성에 구글 API 무료 크레딧을 활용한다.

## 성장 기록
상위 노출 키워드, 전환율 높은 글 패턴, 실패한 니치 등은 learnings/에 기록한다.
