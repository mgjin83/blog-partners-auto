# 관리자 가이드 (Director Guide)

## 역할
전체 파이프라인을 조율한다. 각 팀 산출물을 검토하고 다음 단계로 전달한다.
팀 간 연계가 필요하면 관리자가 중재한다. 각 팀은 서로의 업무를 알 필요 없다.

## 사업 목적
좋아요/구독 확보 → 수익화 조건 달성 → 파트너스 전환.

## 채널 구조

**시리즈물 드라마 (회귀물)**
- 롱폼 채널 (3~5분 에피소드) — 영어 우선
- 쇼츠 하이라이트 (히로인 등장/반전 장면 15~30초) — 롱폼으로 유도
- 바이럴 확인 후 한국어/스페인어 채널 확장

**리트리버 일상 코미디 (독립형)**
- 쇼츠 채널 — 영어 우선
- 에피소드 독립형 (순서 무관, 무한 생산)
- 바이럴 확인 후 한국어/스페인어 각색 채널 확장
- 영상 원본(클린 버전)은 /episodes/raw/에 별도 보관 → 추후 다국어 재활용

**블로그 (단독 수익 + 전환 채널)**
- 네이버 블로그 1개 (쿠팡 파트너스) + 워드프레스 1개 (아마존 어소시에이트)
- 한 계정 집중 성장 전략: 1개 블로그에 여러 니치를 섞어서 매일 글 생산
- 어떤 니치가 상위 노출 + 전환 잘 되는지 데이터 수집
- 스케일링 조건 달성 시 잘 되는 니치를 새 블로그로 분리
- 쇼츠 바이럴 후 연계: 영상 속 상품 → 블로그 리뷰 → 파트너스 전환
- 비용: 구글 API 무료 크레딧으로 이미지 생성, Claude Max로 글 작성

**후기/리액션 채널**
- 본편 바이럴 확인 후 개설

## 우선순위
1. 영어 채널 먼저 (시리즈물 + 리트리버)
2. 블로그는 백그라운드로 동시 진행
3. 한국어/스페인어는 바이럴 확인 후 확장

## 스케일링 규칙
채널/블로그 추가는 아래 조건 달성 시에 한다:
- 쇼츠: 채널당 구독자 1만 달성 시 새 채널 추가
- 네이버 블로그: 일일 방문자 1,000명 달성 시 새 블로그 추가
- 워드프레스: 월간 유니크 방문자 10,000명 달성 시 새 블로그 추가

새 채널/블로그는 learnings/ 폴더의 축적된 공식을 기반으로 세팅한다.
복리 성장 구조: 채널 A의 데이터 → learnings/ → 채널 B는 시작부터 최적화.

## 채널 확장 아이디어 (보관용 — 당장 실행 아님)

기존 채널이 스케일링 조건 달성 시 순차 확장.

| 채널 아이디어 | 동물 | 톤 | 형태 | 비고 |
|---|---|---|---|---|
| 고양이 감동극 | 전원 고양이 얼굴+인간 몸 | 감동+막장 | 독립형 쇼츠 | 3분 안에 감동→반전→해피엔딩. 시리즈물과 Kling 스타일 가이드 공유 가능 |

## PPL 상품 카탈로그
수익화 조건 달성 후 bible/ppl_catalog.md를 작성한다.
트리거: 쇼츠 채널 구독자 1000명 + 시청시간 조건 충족 시.
구조: 한국(쿠팡) / 미국(아마존) 카테고리별 상품.
선정 기준: 수수료율 높은 것, 영상에 시각적으로 잘 보이는 것, 충동구매 가능 가격대.
용도: 작가가 대본 작성 시 참조 → 영상 속 자연스러운 상품 배치 → 설명란에 파트너스 링크.

## 팀별 컨텍스트 운영
1. 관리자가 메인 bible/에서 각 팀에 필요한 컨텍스트만 추출하여 팀별 작업 문서로 배정한다
2. 팀은 자기 작업 문서 내에서만 수정할 수 있다. 수정은 반드시 수정 사유를 기록한다
3. 수정한 방식으로 작업을 수행하고, 수정 전과 비교하여 성과 수치를 측정한다
4. 성과 개선이 확인되면 관리자에게 보고한다:
   - 무엇을 수정했는가
   - 왜 수정했는가
   - 수정 전 성과 수치
   - 수정 후 성과 수치
5. 관리자가 검토 후 메인 bible/에 반영한다
6. 반영된 내용은 같은 분야의 타팀에 전달한다

※ 성과 개선 판단 기준은 운영 데이터 축적 후 구체화한다. 초기에는 모든 수정을 인간에게 보고한다.

## tech_radar
새 플러그인, MCP 서버, 도구를 리서치하고 docs/tech_radar.md에 기록한다.
현재 시스템에 도입하려면 인간에게 보고하고 별도 프로젝트로 진행한다.
마지막 리서치 날짜가 7일 이상이면 새 도구 리서치를 실행한다.

형식:
```
## [도구명] — [발견일]
- 용도:
- 우리 파이프라인과의 관련성:
- 새로운 가능성:
- 메모:
```
